### Task 1.2 — P0-3 PluginRegistry.instance() 호출 수정

**파일**: `dist_dosa/src_v2/workers/healer_worker_v2.py:990-1000`

**현재 (오류)**:
```python
try:
    from ..core.plugin_registry import PluginRegistry
    reg = PluginRegistry.instance()  # ← AttributeError, except가 swallow
    for k in (f"rule.{name}.enabled", f"rule.{rk}.enabled" if rk else ""):
        if k:
            try:
                reg.set_param(k, bool(on))
            except Exception:
                pass
except Exception:
    pass
```

**확인 사실**:
- `dist_dosa/src_v2/core/plugin_registry.py:56` `class PluginRegistry`는 **모든 메서드가 `@classmethod`** (싱글턴 인스턴스 없음).
- `set_param` 도 classmethod (line 169 부근, 추가 검증 필요).
- 따라서 `PluginRegistry.instance()` 는 AttributeError. 외곽 `except Exception: pass` 가 silently swallow → **set_skill_enabled 호출이 PluginRegistry 에 절대 반영 안 됨**.

**변경**:
```python
try:
    from ..core.plugin_registry import PluginRegistry
    for k in (f"rule.{name}.enabled", f"rule.{rk}.enabled" if rk else ""):
        if k:
            try:
                PluginRegistry.set_param(k, bool(on))
            except Exception as e:
                log.warning("set_skill_enabled set_param fail key=%s err=%s", k, e)
except Exception:
    log.exception("set_skill_enabled PluginRegistry write fail")
```

- 외곽 except 도 `log.exception` 으로 바꿔 향후 silently swallow 재발 방지.
- 동일 패턴 다른 위치에도 있으면 함께 수정 (grep `PluginRegistry.instance` 전수).

**검증**:
- 토글 1회 후 `PluginRegistry.get_param("rule.금강불체.enabled")` 또는 `"rule.geumgang_enabled.enabled"` 가 반영됐는지 단위 검증.
- 로그에 `[SET-SKILL] PluginRegistry write` 류 trace 남기는 단순 보강.
