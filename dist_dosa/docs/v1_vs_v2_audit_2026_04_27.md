# v1 vs v2 감사 보고서 (2026-04-27)

> 이전 audit (`src_vs_src_v2_code_only_audit.md`) 가 짚은 5개 보강 항목을 코드 실측으로 fix 한 결과 정리.

## 0. 본 audit 가 잡은 5일 누적 v2 회귀 (root cause 별)

5일간 v2 마이그 회귀가 누적된 **공통 패턴 4가지**:

1. **dict copy 패턴** — v1 ref 공유 의미를 v2 가 dict copy 로 변경 → 갱신 무시
2. **start 후 sync 패턴** — v1 은 start 전 모든 cfg 박는데 v2 는 start 후 sync → race 윈도우에 default 값 fire
3. **adapter wiring 누락** — v1 SoR 의 set_control_handler / set_target_skills / set_xp_region 같은 wiring 을 v2 facade 가 빼먹음
4. **운영 예외 보정 코드 단순화** — v1 의 stall edge / row IP 매칭 / port race 회피 같은 "지저분한 보정" 을 v2 가 깔끔함 명목으로 누락

## 1. audit 5.1 — coord_tol 변경 경로

### v1 동작
`integration_tick.py:209-216` 가 파력무참 buff active 시 `rule_cfg["coord_tol"] = 1` 갱신.
muscle/main_loop 가 매 tick `cfg.get("coord_tol", ...)` 로 read.
**v1 muscle 은 같은 cfg dict ref 사용 → 즉시 반영.**

### v2 회귀
- `src_v2/muscle/main_loop.py:660`: `self.cfg = dict(cfg or {})` ← **dict copy**
- `src_v2/workers/healer_worker_v2.py:278`: `MainLoop(... cfg={"combat_band": ...})` ← **별도 dict 생성**
- 결과: integration_tick 의 갱신이 muscle 에 영원 반영 안 됨 → 파력무참 buff 동안 coord_tol=1 강제 무용 → 격수 좌표 따라가기 정밀도 저하

### Fix
- `muscle/main_loop.py:660` → `self.cfg = cfg if cfg is not None else {}` (ref 보관)
- `healer_worker_v2.py:278` → `cfg=self.cfg.rule_cfg` 직접 전달, combat_band 도 rule_cfg 에 setdefault

### 검증
파력무참 buff active 시점 → integration_tick 가 rule_cfg["coord_tol"]=1 set → muscle 다음 tick 즉시 1 read.

## 2. audit 5.2 — UDP stall/resume edge 로깅

### v1 동작
`src/workers/healer_worker.py:144-145, 1749-1774`:
- `_udp_stalled = False`, `_udp_stall_since = 0.0` 인스턴스 멤버
- 매 tick atk.seq 비교
  - seq alive → 이전 stall 이면 `[UDP-RESUME] seq=N stall_dur=X.Xs` 로그 1회
  - seq same + 3초 초과 + 미stall → `[UDP-STALL] seq=N 3초+ 동일` 로그 1회
- edge 기반 1회씩만 로그 → 운영자가 "끊겼다 돌아옴" 명확히 인지

### v2 회귀
`src_v2/eyes/udp_watcher.py` 는 `[NO-STATE] 격수 State 미수신` 10초/1회만. **stall/resume edge 추적 없음** → 운영 가시성 저하.

### Fix
`udp_watcher.py` 에 v1 1:1 추가:
```python
self._udp_stalled = False
self._udp_stall_since = 0.0
self._last_seq = 0

# _tick 내부
seq_alive = int(latest.seq) > int(self._last_seq)
if seq_alive:
    if self._udp_stalled:
        self._emit(f"[UDP-RESUME] seq={latest.seq} stall_dur={...:.1f}s")
        self._udp_stalled = False
    self._last_seq = int(latest.seq); self._udp_stall_since = now
else:
    if not self._udp_stalled and int(latest.seq) > 0 and (now - self._udp_stall_since) > 3.0:
        self._emit(f"[UDP-STALL] seq={latest.seq} 3초+ 동일 — 격수 송신 중단?")
        self._udp_stalled = True
```

### 사용자 신고와 매칭
사용자: "정지/재시작 시 격수좌표 못 받아옴" → root cause 는 별도 (audit 5 별 항목 참고)지만 stall/resume edge 가 있어야 운영자가 즉시 진단 가능.

## 3. audit 5.3 — attacker cd report row/IP 매칭

### v1 동작
`src/workers/heartbeat.py:68-80`:
```python
row_idx = int(getattr(rep, "src_idx", 0))
src_ip = ""
if src_addr and isinstance(src_addr, tuple) and src_addr[0]:
    src_ip = str(src_addr[0])
    peers = list(getattr(cfg.net, "peers", []))
    for i, p in enumerate(peers):
        if str(p).strip() == src_ip:
            row_idx = i
            break
```
peers IP list 와 src_addr 매칭 → **다중 힐러 환경에서 healer_idx 설정 오류 방어**.

### v2 회귀
`src_v2/workers/attacker_worker_v2.py:391-394`:
```python
row_idx = int(getattr(rep, "src_idx", 0))  # reported 그대로
src_ip = ""
if src_addr ...: src_ip = str(src_addr[0])  # src_ip 기록만, row 매칭 X
```
**peers 매칭 누락** → 두 힐러가 같은 healer_idx (cfg 오류) 로 송신 시 같은 row 에 덮어쓰는 사고.

### Fix
- `attacker_worker_v2.py:_handle_cd_report` 에 v1 동치 매칭 로직 추가 (peers 우선, reported_idx fallback)
- `attacker_worker_v2.py:set_peers(list)` 메서드 신설
- `v1_compat.py:_build_and_start_v2` (attacker) 에서 `self._v2.set_peers(cfg.net.peers)` 호출 + `[atk-v2] peers 주입 [...]` 로그

## 6. 본 audit 의 부수 fix (사용자 신고 직접 매칭)

audit 5 항목 외에 사용자가 신고한 root cause 들 중 **v2 마이그 회귀** 로 확정된 것:

### 6.1 UDP bind port race (사용자 "정지/재시작 시 격수좌표 못 받음")
- 로그 증거: `[v2][!] udp bind 10회 실패 — WinError 10048`
- root cause: ControlListener 가 같은 port 점유 → bind 1초 grace 부족
- Fix: bind 30회 × 200ms (6초) 재시도, `_stop_ctrl_listener` wait 3000ms + socket 명시 close

### 6.2 ControlCmd handler 미등록 (사용자 "격수에서 힐러 stop 안 됨")
- root cause: `RealUdpAdapter` wrapper 가 `UdpReceiver.set_control_handler` 호출 안 함 → 워커 활성 시 격수 ControlCmd 무시
- Fix: facade `_build_adapters` 에서 `_r.set_control_handler` wiring + `cmd_received` Qt signal + `_handle_remote_cmd_active` 핸들러

### 6.3 RuleContextBuilder dict copy (사용자 "체크 안 한 스킬도 시전")
- v1 ref 공유 → v2 dict copy → set_skill_enabled 갱신 무시
- Fix: `decision.py:55` `dict(cfg or {})` → `cfg if cfg is not None else {}`

### 6.4 facade start 후 cfg sync race (사용자 "시작 시 전체 다 시전")
- v1 은 start 전 cfg 다 박음. v2 는 start 후 sync → race 윈도우에 default rule_cfg (전부 enabled=True) 로 fire
- Fix: `v1_compat.py:_build_and_start_v2` cfg setter 들을 `_v2.start()` **이전에** 호출

### 6.5 ready_prev edge 의존 (사용자 "백호 1번 시전 후 영원 안 씀")
- v2 룰: cd>0 한 번이라도 OCR 잡아야 ready_prev=False reset → cd OCR 미관측 환경에서 한 번 fire 후 영원 차단
- Fix: ready_prev edge 폐기, v1 SkillSpec 동치 `last_cast + cooldown_sec(5s)` 게이트

### 6.6 watcher event-driven 가정 (사용자 "공증 임계 넘었는데 안 씀")
- v2 watcher: 값 변할 때만 publish → 시작 시 이미 임계치 아래면 룰 평가 영원 안 일어남
- Fix: cooldown_watcher 빈 result 도 publish, hpmp_watcher 매 tick publish (polling)

### 6.7 메인 키보드 VK 송신 (사용자 "스킬 시전해도 게임이 안 받음")
- v2 `VK_DIGIT` 가 0x30~0x39 (메인 키보드) → 게임이 NumPad 만 받음
- Fix: `tap_numpad` helper 신설, v1 `press_normal_vk` (NumPad VK → 메인 변환 + keybd_event) 위임

### 6.8 skill_enabled key_map 충돌
- `set_skill_enabled` key_map 의 "금강불체" → "boho_enabled" (잘못, "보호" 와 충돌). 미해결 — 별도 turn.

### 6.9 격수 cd/buff target_skills default healer
- 격수 워커가 default 로 healer 스킬 (파력무참/백호의희원) 추적 → `[CD-OCR-MISS]` 폭주
- Fix: `set_own_skill_names` 가 cd 영역에 격수 클래스 스킬, buff 영역에 v1 SoR 동치 ["혼마술","무장","보호"] 분리 set

### 6.10 진입점 role 강제 → 사용자 라디오 무시
- 첫 fix `_forced_initial_role` 이 사용자 라디오 변경 매번 무시
- 최종 fix: forced 폐기, entry default + 디스크 role 우선

### 6.11 격수 → 힐러 송신 동적 IP 학습 누락
- v1 `recv.last_src_addr()` 동적 학습 vs v2 cfg.peers static (도사 PC 가 같은 cfg 쓰면 자기 자신으로 송신)
- Fix: `_UplinkSenderShim.set_attacker_addr` + `_on_attacker_seen` worker 전달

### 6.12 worker 미시작 시 _send_ctrl 차단
- v2 `_send_ctrl` 가 worker None 시 무조건 skip → 격수 GUI 띄우자마자 "전체 시작" 누르면 송신 0
- Fix: worker 있으면 위임 (Attacker class 한정), 없으면 직접 socket 만들어 ControlCmd unicast

### 6.13 send_control facade no-op
- `AttackerWorkerV1Facade.send_control` 이 sender.send_control 위임 시도 → `RealUdpSenderAdapter` 에 그 메서드 없어 항상 False
- Fix: v1 `attacker_worker.send_control` 1:1 포팅 (`_sender.send_to(peer, port, ControlCmd.to_bytes())`)

### 6.14 _start_healer GUI peers/port 미반영
- attacker `_start_attacker` 는 GUI peers → cfg 반영하는데 healer 만 누락 → uplink_sender 미생성
- Fix: `_start_healer` 첫 줄에 동일 갱신 + 진단 로그

### 6.15 로그 파일 분산 (사용자 "여러 파일 만들지 마")
- 워커 stop/start 마다 `_setup_compat_logger` 가 새 `healer_v2_{ts}.log` 생성
- Fix: module-level singleton 으로 첫 호출 시 1회만 파일 생성, 이후 재사용
