# P0-1 — UDP 수신 경로 안정화 (2026-04-28)

## 1. 문제

**증상 (재현 조건)**: 힐러 워커 stop → 즉시 start 시
- `[v2][!] udp bind 30회 실패 — 격수 좌표 수신 불가: <OSError>`
- `[UDP-RECV] adapter=None — udp 비활성`
- attacker State 입력이 통째로 사라짐

## 2. v1 SoR

`src/net/udp_receiver.py`:
- L21: `self._sock.bind((bind_host, port))` 즉시 bind, 실패 시 raise
- L23-25 주석: SO_REUSEADDR 금지 (Windows UDP TIME_WAIT 없음, ControlListener 공존 시 라우팅 비결정)
- L107-112 `stop()`: `_running=False` + `_sock.close()`

v1 동작: 워커 stop 시 socket close → 다음 start 가 동일 포트 즉시 bind 가능.

## 3. v2 현재 상태 (root cause 확정)

### 3.1 호출 체인
1. `HealerWorkerV2.stop()` (`workers/healer_worker_v2.py:760`) → `for w in (..., udp, ...): w.stop(timeout)`
2. `UdpWatcher.stop` → `BaseWatcher.stop` (`eyes/base_watcher.py:69-72`)
3. **`BaseWatcher.stop` 은 폴링 thread 만 join, `self.adapter.stop()` 미호출**
4. `RealUdpAdapter._r` (`UdpReceiver`) 가 살아있는 상태로 `out["udp"]=None` 만 facade 가 정리
5. process 안에서 이전 `UdpReceiver._sock` 이 port 51900 보유 중
6. 다음 start: `build_healer_adapters` 가 `RealUdpAdapter(port=cfg.net.port)` 30회 시도 → 모두 OSError → `out["udp"]=None`

### 3.2 재현 메모
```
세션 A: 워커 start
  → RealUdpAdapter() → UdpReceiver bind ok port=51900
  → recv thread alive

[사용자] stop 클릭
  → HealerWorkerV2.stop()
  → udp watcher .stop() — 폴링 thread만 종료
  → UdpReceiver._sock 그대로 살아있음 (close() 안 됨)

[사용자] start 클릭 (즉시)
  → build_healer_adapters → RealUdpAdapter(port=51900)
    → UdpReceiver(__init__) → bind(port=51900) → OSError [WinError 10048]
  → except → time.sleep(0.2) → 30회 반복 → 모두 실패
  → out["udp"] = None
  → [UDP-RECV] adapter=None — udp 비활성
```

## 4. 수정 내용 (fixed-in-code)

### 4.1 `src_v2/eyes/base_watcher.py:69-86` 수정

```python
def stop(self, timeout: float = 2.0) -> None:
    # P0-1 fix 2026-04-28: adapter.stop() chain 강제.
    self._stop_evt.set()
    if self.is_alive():
        self.join(timeout=timeout)
    ad = getattr(self, "adapter", None)
    if ad is not None:
        for m in ("stop", "close"):
            fn = getattr(ad, m, None)
            if callable(fn):
                try:
                    fn()
                except Exception:
                    log.exception("watcher %s adapter.%s fail",
                                  self.watcher_name, m)
                break
```

### 4.2 효과
- `RealUdpAdapter.stop()` (`adapters/udp_adapter.py:108-114`) 가 `UdpReceiver.stop()` 호출 → `_sock.close()` 즉시 → 다음 bind 가능.
- 다른 watcher (cooldown / hpmp / yolo / xp) 의 adapter.stop / close 도 동일하게 chain — 잔여 thread 회수.

### 4.3 추가 권장 (별도 PR)
- `RealUdpAdapter.__init__` 의 `r.start()` try/except pass 제거 — start 실패 시 raise 로 승격 (P0-1 §3 지시 4: bind 실패 시 “기능 제한 모드”가 아니라 worker 기동 실패).

## 7. 남은 리스크

1. **외부 프로세스 점유**: 동일 port 를 다른 프로세스가 잡고 있으면 본 fix 로 해결 안 됨. 진단 강화 필요 — bind 실패 시 `netstat` 결과를 로그로 첨부.
2. **ControlListener 공존**: v1 시절 `ControlListener` 가 같은 port 에 별도 bind 했음. v2 는 `UdpReceiver._ctrl_handler` 단일 경로 사용 중 — 회귀 시 race 가능.
3. **socket TIME_WAIT (Linux)**: Windows 에서는 무관하나 Linux 배포 시 SO_REUSEADDR 정책 재검토 필요.
4. **30회 retry 자체 제거 검토**: stop chain 으로 즉시 bind 가능해지면 retry loop 6초 지연이 불필요해짐 → 향후 1~2회로 축소.
