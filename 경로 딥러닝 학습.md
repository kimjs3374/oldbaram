# 경로 딥러닝 학습 (NavBrain) — 설계·운영 문서 (2026-07-05)

> 목표: **지금까지 수집된 로그·맵 데이터로 추종 경로를 학습**하고, 앞으로도
> 계속 수집→계속 학습하는 루프를 상시 가동한다. 막힘 없이 격수를 따라가고,
> 사람(격수)의 발자국을 기보 삼아 **사람처럼 걷는** 것이 목표.
>
> 관련 문서: `맵이동 로직.md`(현행 추종 명세), `맵 데이터화 로드맵.md`(grid 수집 설계).
> 이 문서는 그 위에 올라가는 **학습 계층**의 정본이다.

## 0. 알파고 구조를 옛바 제약에 맞게 옮기면

| 알파고 | NavBrain | 근거 |
|---|---|---|
| 인간 기보 (수백만 국) | 격수 발자국 TRAIL-PUSH 4.7만 스텝 + 1Hz 궤적 8.8만 (2026-07-05 실측) | 격수=사람 조작, golden 데이터 |
| 정책망 (다음 수 예측) | **NavNet**: 로컬 grid 패치 + 목표방향 → 다음 한 칸 (5-way CNN, ONNX) | behavior cloning |
| 가치망+MCTS 탐색 | **NavGraph**: walk/blocked/막힘률 grid 위 Dijkstra flow field | 맵≤500셀이라 완전탐색이 μs |
| self-play 반복 학습 | 세션마다 자동 수집(MapGrid)→클라우드 병합→주기 재학습 | 이미 가동 중인 수집 인프라 재사용 |
| evaluator (새 모델 검증 대국) | **리플레이 평가 게이트**: 실로그 held-out 대비 개선 입증 시에만 배포 | `_nav_replay_eval.py` |

RL(실게임 시행착오)은 안 한다 — 시뮬레이터가 없고 밴 위험. 대신
**모방학습(사람 기보) + 그래프 탐색(누적 관측)** 조합이 같은 효과를 안전하게 낸다.

## 1. 2계층 학습 구조

### 계층 1 — NavGraph (온라인 학습, 매 세션 자동 갱신)

- 입력: 살아있는 `MapGrid`(cells walk/tab, blocked, attempts try/block) — **세션 중에도 계속 자람**.
- 목표 셀 기준 Dijkstra flow field. 간선 비용:
  - 기본 1 + `WALK_BONUS/(1+walk)` (사람이 많이 밟은 통로 선호 = 사람같은 동선)
  - blocked 간선: cnt 기반 페널티 (1회=의심 +4, 3회+=사실상 차단 +50)
  - attempts 막힘률(block/try, try≥3): 비율 비례 페널티 (몹 vs 벽 구분)
- **기권(abstain) 규칙이 핵심** (v113 B3D 3.5배 악화 교훈):
  - 맵 데이터 부족(walk 셀 < 최소치) → None
  - 목표까지 관측 셀로 이어진 경로 없음 → None
  - None이면 호출측은 **기존 trail 로직 그대로** — 악화 불가능 구조.
- 캐시: (맵, 목표셀, grid 버전) 키. grid에 새 관측 들어오면 버전 증가→재계산.

### 계층 2 — NavNet (오프라인 딥러닝, D머신 주기 재학습)

- 문제 정의: **goal-conditioned behavior cloning**.
  샘플 = 격수 trail의 (현재셀 s_t → 다음셀 s_t+1), 목표 = s_t+k (k=3~15 hindsight).
  "사람이 이 지형에서 저 방향의 목표로 갈 때 다음 한 칸을 어디로 딛는가".
- 입력:
  - 패치 15×15×6ch (현재셀 중심): walk(log 정규화), observed, blocked U/D/L/R 증거율
  - 스칼라: 목표 Δ(clamp ±16 정규화), 직전 방향 one-hot(5)
- 출력: 5-way (U/D/L/R/STAY) softmax.
- 모델: conv×3 + fc×2 (digit_cnn 관례), 파라미터 ~10만, ONNX opset12, CPU 추론 <1ms.
- 패치 기반이라 **맵 이름에 무관** — 데이터 적은 맵에서도 지형 패턴으로 일반화.
- 런타임 결합: NavGraph 방향과 NavNet 방향이 **일치할 때 고신뢰**, 불일치면
  신뢰 하향(on 모드에선 기권). 알파고의 정책망×탐색 결합과 동형.

## 5. 리플레이 평가 게이트 (`_nav_replay_eval.py`) — 배포 전 필수

실게임 0회 접촉으로 오프라인 검증. **2026-07-05 배포정합 재설계** — 근거:
런타임 `on` 훅은 trail 부재/STUCK 에서만 개입(기존 동작=직선/부호휴리스틱인
지점뿐). 탁 트인 직선 구간(홀드아웃 실측 66%)의 모방은 trail 추종이 이미
담당(그게 곧 사람 경로)이라 게이트 비대상.

| 지표 | 방법 | 게이트 |
|---|---|---|
| A1. 모방 회귀가드 | held-out 다음 칸 top-1: 그래프+정책망 vs 그래프 단독 | 결합 ≥ 단독 (망이 모방을 깎으면 FAIL) |
| A2. 결정지점(hard) | **직선이 틀리는 지점**(코너/벽, 전체 34%)의 일치율 | 직선 초과 + 그래프 단독 −2pp 이내 |
| B. 도달 시뮬레이션 | 관측 grid 월드모델에서 (출발,목표) 걸어봄 | 도달률 ≥ 직선 (막힘없이 따라가기 핵심) |
| D. 위반율 | 제안이 blocked 간선/미관측 셀로 들어가는 비율 | < 2% |
| (정보) A 전체 vs 직선 | 전체 일치율 | 게이트 아님 (직선 구간은 trail 담당) |
| (정보) C STUCK 리플레이 | 실로그 STUCK 지점 '뚫린 방향' 적중률 | 게이트 아님 — 실행된 정책이 뚫린 방향을 스스로 만든 관찰 데이터(선택편향)라 반사실 평가 불가. shadow 실측으로 대체 |

게이트 FAIL → 배포 금지, 모델 파일 교체 안 함. (알파고 evaluator와 동일 규율.)

2026-07-05 1차 실측 (홀드아웃 최신 5세션 10,678샘플/1,006세그먼트):
- B 도달률: NavBrain 98.9~99.9% vs 직선 85.7% (**+13pp**)
- A2 hard: NavBrain 26.5~30.8% vs 직선 0% (정의상) / 그래프 단독 27.1%
- D 위반율 0.00%. NET_W 스윕(1.5/3/6/10) → **10 채택** (전체 64.8%/hard 28.1%)
- NavNet: val 65.1~65.5% (sqrt weight 최적), torch↔ort 패리티 2.9e-6

### 6.2 `on` 모드 개입 지점 (전부 "기존이 막막한 곳"만, 우선순위 침범 없음)

1. **no-trail 폴백** (B3T에서 trail 없어 직선으로 떨어질 때): NavGraph 경로 우선,
   기권 시 기존 직선. — 기존이 '맹목 직선'인 유일 구간이라 순수 이득.
2. **STUCK-ORTHO 방향 선택**: 직교 2방향 중 flow field 비용 낮은 쪽. 기권 시 기존 부호.
3. **B3 축 스왑 보강**: is_wall 이진 → 방향별 비용 비교 (선택지가 두 축뿐이라 안전).
4. **B2:MAPNEQ 폴백 체인**: exit_dir 다음 순위에 NavGraph(목표=exit_coord) 삽입.

> 2026-07-05: 위 4개 개입점(suggest/unstick_dir) 전부 `avoid=peers` 배선 —
> 다른 봇 좌표를 회피 셀로 넘겨 flow 후보에서 제외(관측 통로만 쓰므로 옆걸음
> 드리프트 없음). §6.5 참조.

### 6.3 불변 안전 규칙 (모델이 절대 못 건드리는 것)

- FORCE-EXIT / WHITETAB-BLOCK / TAB-CONFIRM / MAP-PAUSE 게이트 **위에 못 올라감**
  (개입은 전부 `_decide_move_raw` 내부 깊숙한 분기).
- `map_change_pending` / MAP-JUMP hold 창(맵명 OCR 지연 오염) 중 개입 금지.
- 선비족/라인 exit_dir 고정 테이블(`_sunbi_exit_dir` 등) 대체 금지.
- trail 추종과 모델 제안을 한 틱에 혼합 금지 (진동 방지).
- 제안 방향 0.4s 히스테리시스 (want 급변 → STUCK 감지 무력화 방지).
- PEER-WAIT 불변(옆걸음 금지) — 단 2026-07-05 STUCK 시계 리셋으로 인내대기 강화
  (§6.5-4). 슬롯 추종(stop_before)은 2026-07-01 롤백 상태 — 재도입은 재실증 후.

봇충돌(P2)·빨탭복귀 오판(P3)이 3.5s STUCK-RESET→`note_blocked`로 **가짜 벽**을

"장애물 다 학습"이 **가짜 장애물까지 학습**해 flow/​is_wall을 오염시키는 역효과.

수정 (전부 `_apply_stuck_filter`, §6.3 게이트 아래 분기, 옆걸음/슬롯 재도입 없음):

1. **가짜 벽 영구학습 차단** (P1 핵심): STUCK-RESET에서 원인이 ①재고정 중
   (`_reanchor_active`) ②앞칸/인접이 봇(peers) ③격수와 추종축 >3칸 뒤처짐 —
   이 중 하나면 `note_blocked`(grid 영구) **생략**, 휘발 blacklist(TTL)만 유지.
   로그 `STUCK-RESET … → BL-SKIP(가짜벽)`. 진짜 벽(정렬+근접+peer없음)은 여전히
   BL-ADD. 진짜 벽의 1차 증거는 격수 막힘률(attempts, 격수 실제 키)이 더 신뢰.
2. **재고정/탭복귀 STUCK 유예** (P3): 가드에 `_reanchor_active` /
   `_tab_confirm_active` / `post_tab_grace_active` 추가(흰탭과 동급). 격수에게
   복귀하느라 잠깐 막힌 걸 STUCK-ORTHO로 오판해 삥삥 도는 것 차단
   (실측: 재고정 완료후 6초내 STUCK 56~65% → 억제).
3. **avoid=peers 배선** (P2): §6.2의 suggest/unstick_dir 4곳 전부 peers 전달 →
   NavBrain이 봇 점유 셀을 flow 후보에서 제외(관측 통로만 쓰므로 드리프트 무).
4. **PEER-WAIT 인내대기** (P2): 앞칸이 봇이면 STUCK 시계 리셋 → RESET/가짜벽으로
   절대 승격 안 함. 봇은 곧 비키거나 같이 이동(격수 추종). 옆걸음 금지 유지.
