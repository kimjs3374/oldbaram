# 선비족 사냥 순서 네비게이션 오버레이 (격수 전용) — 설계 구상

> 2026-06-12 초안. 구현 전 설계 문서. 사용자 요구 원문 기반 + 구현 구상.
> 관련: `맵이동 로직.md`(힐러 추종), `src/ui/overlay.py`(오버레이 공통),
> `src/app/hunt_analytics.py`(맵 히스토리 — 본 기능의 y 시퀀스 소스 후보).

---

## 1. 목적 / 범위

- **격수 모드 전용** 인게임 오버레이. 현재 선비족 지역(x)의 굴 배치 지도를 그리고
  **다음에 가야 할 굴(y)을 강조** 표시.
- 굴 순서는 ①수동 입력(텍스트필드) ②추천 순서 유도 ③실주행 학습/확정 3계층.
- 힐러/쩔캐 동작에는 일절 관여하지 않음 (표시 전용, 키 입력 없음).

## 2. 변수 정의 / 획득

| 변수 | 의미 | 획득 |
|---|---|---|
| base | `선비족` 또는 `제2선비족` | map OCR (`Attacker._last.map_name`) |
| **x** | 지역 1~5 | map OCR 자동 + GUI 수동 입력 가능 |
| **y** | 굴 1~5 (이번 기능의 "굴") | map OCR |
| (z) | 채널/층 `(N)` — **본 기능과 무관** (파력무참/지폭 게이트용과 별개) | — |

- 파싱: 굴=`^(제2)?선비족([1-5])-([1-5])(\((z)\))?`, 허브=`^(제2)?선비족([1-5])`
  (x 뒤 -y 없음, '선비족1방' 류 변형 허용). `(z)`는 굴 내 층 — 강조 타이밍 판단에 사용.
- 매칭 안 되는 맵(입구, 타 사냥터)은 **무시** — 진행 중 lap 상태를 깨지 않음
  (입구 경유가 시퀀스를 오염시키면 안 됨).
- **강조 안내 타이밍 (사용자 확정 2026-06-12)**: 다음 굴 강한 강조는
  `선비족x-y(7)에서 나와 허브(선비족x)로 왔을 때` (제2선비족 동일).
  굴 내부 사냥 중에는 다음 굴을 옅은 테두리로만 표시. 트래커가 굴별 마지막
  (z)를 기억(_last_z), 허브 도착 시 z==7이면 from_z7=True → 오버레이 강한
  강조(노랑 채움+▶+'N굴로 이동!') + notice 알림. z<7 채 허브로 나오면
  옅은 표시만 (실수/이동 중 — 잘못된 유도 방지).
- 같은 y 연속 관측은 dedup (맵 OCR 1초 폴링이라 같은 맵 반복 관측됨).
- map OCR은 이미 known_maps 화이트리스트 게이트(v10) + 좌표연속 게이트(v33)를
  통과한 값만 들어오므로 별도 오독 방어 불필요.

## 3. 맵 레이아웃 데이터 (x별)

슬롯 6개 고정 그리드: `TL`(좌상) `TR`(우상) `LM`(좌중) `RM`(우중) `BL`(좌하) `BR`(우하).
오와열 정렬: 컬럼 2개(안쪽 좌/우) + 사이드 2개(좌/우 에지), 로우 3개(상/중/하)의
**고정 좌표 격자**에 박스를 배치 — 텍스트 길이와 무관하게 정렬 유지.

### x = 1
```
┌──4───3──┐
│          │
5          1
│          │
└─입구──2──┘
```
| 슬롯 | TL | TR | LM | RM | BL | BR |
|---|---|---|---|---|---|---|
| 내용 | y4 | y3 | y5 | y1 | 입구 | y2 |

- 네굴 추천: `5→4→3→2` / 다섯굴 추천: `5→4→3→1→2`

### x = 2, x = 3 (동일 레이아웃)
```
┌──5───4──┐
│          │
입구        1
│          │
└──2───3──┘
```
| 슬롯 | TL | TR | LM | RM | BL | BR |
|---|---|---|---|---|---|---|
| 내용 | y5 | y4 | 입구 | y1 | y2 | y3 |

- 네굴 추천: `5→4→3→2` / 다섯굴 추천: `5→4→1→3→2`

### x = 4, x = 5 (동일 레이아웃)
```
┌──4───3──┐
│          │
5          입구
│          │
└──1───2──┘
```
| 슬롯 | TL | TR | LM | RM | BL | BR |
|---|---|---|---|---|---|---|
| 내용 | y4 | y3 | y5 | 입구 | y1 | y2 |

- 네굴 추천: `3→4→1→2` / 다섯굴 추천: `3→4→5→1→2`

### 코드 표현
```python
# hunt_nav.py 모듈 상수
LAYOUTS = {  # x -> {slot: label}, label: 1~5(굴) or 0(입구)
    1: {"TL": 4, "TR": 3, "LM": 5, "RM": 1, "BL": 0, "BR": 2},
    2: {"TL": 5, "TR": 4, "LM": 0, "RM": 1, "BL": 2, "BR": 3},
    3: {"TL": 5, "TR": 4, "LM": 0, "RM": 1, "BL": 2, "BR": 3},
    4: {"TL": 4, "TR": 3, "LM": 5, "RM": 0, "BL": 1, "BR": 2},
    5: {"TL": 4, "TR": 3, "LM": 5, "RM": 0, "BL": 1, "BR": 2},
}
RECOMMEND4 = {1: [5,4,3,2], 2: [5,4,3,2], 3: [5,4,3,2],
              4: [3,4,1,2], 5: [3,4,1,2]}
RECOMMEND5 = {1: [5,4,3,1,2], 2: [5,4,1,3,2], 3: [5,4,1,3,2],
              4: [3,4,5,1,2], 5: [3,4,5,1,2]}
```
- base(선비족/제2선비족)는 레이아웃 동일 가정 (x만으로 결정).

## 4. 순서 결정 — 3계층 우선순위

```
수동 입력(텍스트필드, 사용자 편집) > 학습 확정 > 추천 유도 > (없음: 강조 없이 지도만)
```

### 4-1. 수동 입력
- 격수 패널 텍스트필드 `굴 순서` — `"5,4,3,2"` 형식. 유효 = 1~5 중복 없는 3~5개.
- **동작 중 변경 즉시 반영** (textChanged → tracker.set_manual_order).
- 세굴(`4,3,2`)·네굴·다섯굴 모두 그대로 순환: next = order[(idx+1) % len].

### 4-2. 학습 (텍스트필드 미입력/자동입력 상태)
상태: `visited` = 이번 바퀴 관측 y 시퀀스(중복 없는), `confirmed` = 확정 사이클.

1. 새 y 관측, y ∉ visited → visited.append(y). 텍스트필드에 visited를
   **잠정값으로 자동 입력** + 안내("순서 학습중: 5,4,3,2").
2. 새 y 관측, y ∈ visited → **바퀴 경계**. 확정 규칙(회전):
   `confirmed = visited[idx:] + visited[:idx]` (idx = visited.index(y)).
   새 바퀴 visited = [y]로 리셋. 텍스트필드 자동 입력 + 안내("순서 확정: …").

   | 관측 | 재방문 | 확정 |
   |---|---|---|
   | 5,4,3,2 → 5 | idx0 | `5,4,3,2` |
   | 5,4,3,2,1 → 4 | idx1 | `4,3,2,1,5` |
   | (확정 4,3,2,1,5 상태) 4,3,2,1 → 4 | idx0 | `4,3,2,1` (5 탈락) |

3. **확정 후에도 매 바퀴 재확정** — 다음 바퀴 경계마다 그 바퀴의 visited로 갱신
   (위 표 3행이 근거). 단 §4-4 자동/수동 구분 참조.

### 4-3. 추천 유도 (학습 단축 경로)
- 미확정 + visited 첫 y == 해당 x 추천 시작굴이면 → **다섯굴 추천**을 잠정 안내
  (네/다섯굴 추천이 시작굴 동일 → 사용자 예시 `선비족1 첫 1-5 → 5,4,3,1,2 유도`
  대로 다섯굴 기본).
- 잠정 안내 중에도 visited 누적은 §4-2 그대로 — 2번째 바퀴 첫 y 재방문 시점에
  실주행 기준으로 세/네/다섯굴 확정:
  - `5,4,3,5` → `5,4,3` 세굴 / `5,4,3,2,5` → 네굴 / `5,4,3,2,1,5` → 다섯굴.
- 잠정 안내 중 추천 경로 이탈(예: x1에서 3 다음 1 대신 2) → 강조는 현재 y 기준
  추천 내 다음으로 추종, 확정은 어차피 바퀴 경계의 visited가 결정.

### 4-4. 자동입력 vs 사용자 수정 구분 (충돌 방지 핵심)
- 텍스트필드 쓰기 주체 플래그: 프로그램 자동 입력 시 `_auto_filled=True`
  (blockSignals로 재진입 차단). 사용자가 직접 타이핑하면 `_auto_filled=False`.
- `_auto_filled=True` 상태 = 학습 계속 → 매 바퀴 재확정으로 필드 갱신.
- 사용자가 수정한 순간 = **수동 고정** → 학습이 필드를 덮어쓰지 않음.
  필드 비우면 다시 학습 모드 복귀.

### 4-5. 확정 상태에서 불규칙 주행 ("최대한 순서에 맞게 유도")
- 현재 y ∈ order → next = order[(idx+1) % len]. (순서를 벗어나 중간으로 점프해도
  그 지점 기준 다음 굴을 안내 — 자연 재동기화.)
- 현재 y ∉ order (확정 순서 밖 굴 진입) → 직전 강조 유지 + 회색 보조표시로
  "순서 밖" 안내. 확정은 변경하지 않음 (불규칙 1회로 사이클 파괴 금지).
  단 학습 모드(자동입력)면 visited에는 누적되어 다음 바퀴 경계에서 자연 반영.

### 4-6. x 전환
- x가 바뀌면 (다른 지역 이동): 레이아웃 교체 + visited 리셋.
- 확정/수동 순서는 **x별 세션 캐시** `{x: order}` — 되돌아오면 복원.
  텍스트필드도 해당 x의 캐시로 교체 표시.

## 5. 오버레이 UI 설계 (`HuntNavOverlay`)

- `_ScaledOverlay` 상속 — 드래그/위치저장(키 `"huntnav"`)/투명도/편집모드/msw-shift
  전부 기존 5종 오버레이와 동일 배선 (v52 HuntOverlay 추가 때와 같은 6곳 배선).
- 격수 모드 + 오버레이 ON일 때만 show (helper/hpmp/hunt와 동일 규칙).

### 렌더링 (오와열 정렬)
```
폭 W(기본 220px, 해상도 스케일). 격자 좌표(비율 고정):
  colL=0.08W  col1=0.32W  col2=0.68W  colR=0.92W
  row0=top  row1=mid  row2=bottom (행 높이 동일)
노드 박스: 고정 크기(예 44x26 스케일), 중심을 격자점에 정렬 → 오와열 보장.
링: 슬롯 중심을 잇는 라운드 사각 경로 (배경 라인).
```
- 노드 표기: `1`~`5` + `입구`.
- 색: **다음 굴 = 강조**(노랑 채움 + 굵은 테두리 + `▶` 마커), 현재 굴 = 녹색
  테두리, 그 외 순서 포함 굴 = 회색, 순서 밖 굴 = 어두운 회색, 입구 = 청회색.
- 상단 타이틀: `선비족1 네비` (base+x. 제2선비족이면 `제2선비족1 네비`).
- 하단 텍스트 2줄:
  - `순서: 5→4→3→2 (확정)` — 상태 라벨: 수동/학습중/추천/확정.
  - `다음: 3굴` (+ 순서 밖이면 `현재 1굴은 순서 밖`).

### GUI (격수 패널)
- `굴 순서` QLineEdit (placeholder "예: 5,4,3,2 — 비우면 자동 학습").
- `x` 표시 라벨(OCR 자동) + 수동 override 스핀(0=자동).
- `네비 오버레이` 체크박스 (chk_overlay와 별개 세부 토글 — helper처럼).
- 설정 저장: 순서 텍스트(수동 입력분만), x override, 토글, 위치("huntnav").

```

        │                            │ (순수 로직, attacker 스레드)
        └─ get_hunt_nav_snapshot() ←─┘  {x, layout필요없음(오버레이가 보유),
                                         order, order_state, next_y, cur_y,

MainWindow._tick_analytics (기존 1s 타이머에 합류)
        ├→ HuntNavOverlay.update_nav(snap)
        ├→ 텍스트필드 자동 입력 (auto_fill_text 변화 시, blockSignals)
        └→ 안내 push_alert (notice 변화 시, SkillAlertOverlay 재사용)
텍스트필드 textChanged ──> worker.set_cave_order_text(text, manual=True)
```

| `src/app/attacker.py` | tracker 보유, 맵변경 시 observe, snapshot 제공, set_cave_order |

| `src/ui/hunt_nav_overlay.py` **신설** | `HuntNavOverlay(_ScaledOverlay)` 렌더 |
| `src/ui/main_window.py` | 격수 패널 위젯 + 오버레이 6곳 배선 + tick 라우팅 + 자동입력/안내 |
| `src/ui/settings_io.py` | 순서 텍스트/x override/토글 저장·복원 |

## 7. 엣지 케이스

1. **입구/비매칭 맵 경유**: observe 무시 — visited 유지 (바퀴 중 입구 통과 허용).
2. **맵 OCR 일시 오독**: known_maps+좌표 게이트 통과값만 수신. 그래도 같은 바퀴에
   비정상 y 1회 유입 시 — 학습엔 들어가지만 다음 바퀴 경계에서 자연 교정됨.
3. **(z) 채널 변경**: x-y 동일하면 y 변화 아님 → dedup으로 무시.
4. **세굴 미만(2굴 이하) 사이클**: 확정 최소 길이 3 미만이면 확정 보류(잠정 유지).
5. **재시작 복원**: 수동 입력 순서만 설정 파일 저장. 학습 확정은 세션 한정
   (사냥터 바뀌는 일상 고려 — 캐시는 x별 세션 내만).
6. **제2선비족**: x 파싱 동일, 레이아웃 동일 가정. 타이틀만 base 반영.

## 8. 구현 전 확인 질문 (사용자)

1. **자동입력 후 사용자가 필드를 직접 수정하면** 학습 중단(수동 고정)이 맞는지?
   비우면 학습 재개로 할 예정. (§4-4)
2. **x 전환 시** 이전 x에서 확정한 순서를 세션 내 기억했다가 복원하는 방식 OK?
3. **제2선비족 레이아웃**이 선비족과 동일한지? (다르면 x별 표 추가 필요)
4. 오버레이 **신규 토글 체크박스**(네비 ON/OFF) 별도로 둘지, 기존 `오버레이`
   체크 하나에 묶을지? (제안: helper처럼 묶고 위치만 별도 드래그)
5. 순서 밖 굴 진입 시 "직전 강조 유지 + 순서 밖 표기" 방식 OK? (§4-5)
