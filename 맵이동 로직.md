# 맵이동 로직 (2026-04-16 기준)

힐러가 격수를 따라 맵을 건너는 전체 파이프라인. 비전(OCR+YOLO) + UDP만 사용.
메모리/미니맵 접근 없음.

## 0. 구성 요소

| 모듈 | 파일 | 역할 |
|---|---|---|
| 격수 | `attacker.py` | 인간 조작. UDP로 `(map_name, map_seq, x, y, coord_valid)` 송신 (30Hz) |

## 1. 입력 신호

매 프레임 (≈20fps) 다음이 수집됨:

| 신호 | 소스 | 용도 |
|---|---|---|
| `atk.map_name`, `atk.map_seq` | UDP | 격수의 현재 맵과 전환 시퀀스 |
| `atk.x, atk.y, coord_valid` | UDP | 격수 좌표 |
| `healer_map` | 맵 OCR | 힐러 자신의 맵이름 |
| `healer_coord` | 좌표 OCR | 힐러 자신의 좌표 |
| `red_raw`, `white_raw` | YOLO | 빨탭/흰탭 1프레임 감지 (bbox conf≥0.25) |
| `map_neq` | 계산 | `healer_map != atk.map_name` |

## 2. MAPCHG 판정과 3단계 전환

### 2.1 격수 맵전환 감지 (`MAP-SEQ-EDGE`)

`controller.update()` 내에서 `atk.map_seq > _last_seen_map_seq` 발생 시:
1. `_pause_until = now + 0.25s` → 힐러 키 release 0.25초.
2. `_map_sync_until = now + 1.5s` → MAP-SYNC 창 개방.
3. `_last_mapchg_ts = now` 기록.
4. 활성 TAB-CONFIRM route 있으면 취소 (`[TAB-CONFIRM-CANCEL]`).

### 2.2 MAP-SYNC (맵 OCR 지연 흡수)

맵 OCR은 CPU로 0.5~1s 지연. 좌표 OCR은 이미 새 맵 기준으로 나옴 → "좌표는 새 맵
/ 맵이름은 이전 맵" 불일치 창 발생. 이 구간 방향키 입력 보류(최대 1.5s).
`h_map == a_map` 되면 즉시 해제 (`[MAP-SYNC-DONE]`).

### 2.3 CTRL-MAPCHG (격수 맵 전환 확정)

`atk.map_name` 갱신 시 controller는 `exit_coord`, `exit_dir`, `old_trail_tail`을
기록. 이후 힐러는 `exit_dir`을 기준으로 맵 전환 방향을 추정.

### 2.4 HMAP-DEBOUNCE (맵 OCR flicker 방어)

`healer_map`이 직전 값으로 되돌아갈 때(2초 내 A→B→A), 3프레임 연속 확인 후 수락.
OCR 한 프레임 튀는 거로 pause/sync 폭주하지 않게.

### 2.5 HMAP-COORD-MISMATCH (좌표 기반 crosscheck)

새 `healer_map` 수락 전, 해당 맵의 trail bbox 밖에 좌표가 있으면 거부
(`[HMAP-COORD-MISMATCH] reject`). bbox 안으로 들어와야 수락.

### 2.6 ATK-COORD-JUMP (격수 좌표 튐 방어)

같은 `atk.map_name`에서 `atk.x/y`가 물리적으로 불가능한 거리로 점프 시
`coord_valid=False` 강제. 보통 격수가 이미 새 맵으로 넘어갔는데 `map_seq`가 아직
안 온 1초 gap에서 발생.

## 3. 힐러 상태 머신 (FSM)

```
IDLE → FOLLOW → COMBAT ⇄ (EMERGENCY, STUCK, DEAD_RECKON)
           ↓
         ENTER_PORTAL → LOADING → NEW_MAP → (COMBAT/FOLLOW)
```

| 상태 | 조건 |
|---|---|
| FOLLOW | 격수 이동 중, 빨탭 없음 |
| COMBAT | 빨탭 감지 (격수 위치 확정) |
| ENTER_PORTAL | `map_seq` edge 감지 직후 |
| LOADING | 힐러 맵 전환 중 |
| NEW_MAP | 전환 직후 안정화 |
| STUCK | 오래 같은 좌표, 격수 좌표 stale |
| DEAD_RECKON | 좌표 일시 유실 시 이전 방향 유지 |
| DEAD | HP 0 또는 유령 상태 |
| DISCONNECTED | UDP stale 3초+ |

## 4. 이동 결정 (`_decide_move`)

우선순위 순서:

1. **B0 스킬 블록**: 자힐 등 skill 시전 중이면 이동 중단.
2. **B1 DEAD/DISCONNECTED**: 이동 안 함.
3. **B2 MAP-NEQ**:
   - 격수 맵이 힐러 맵과 다를 때. 힐러 맵의 trail을 **끝까지 추종**
     (`progress` < `len-1` 이면 다음 waypoint, 아니면 `exit_dir`).
   - trail 끝 도달 + `map_neq=True` 지속 시 `STAY(past)` → 2초 후 `past_unstick`
     (`exit_dir`로 강제 push).
4. **B3 coord**: 같은 맵에서 격수 좌표와의 차분 `d=(dx,dy)` 기준:
   - `|d| > 2` → 해당 방향 hold.
   - `|d| ≤ 2` → `B3a:tol` release.
5. **B5 a_invalid**: 격수 좌표 invalid면 `last_dir` 기준 맹목 이동.

### 4.1 EXIT-INHERIT (연결된 맵 복귀 방향 반전)

`trail_len == 1`인 짧은 맵은 자체 방향이 부족 → 이전 맵의 `exit_dir`을 상속.
단, A→B→A 패턴(돌아왔다)에서는 **방향을 반전**:
`{L↔R, U↔D}`. B로 갔다가 A로 되돌아왔으면 A에서 원래 방향의 반대로 나가야
함.

### 4.2 KEY-REHOLD (좌표 stale 시 재송신)

같은 방향을 1.5~2초 hold했는데 `healer_coord`가 변하지 않으면 keybd 재송신
(드라이버 레벨 키 상태 리셋).

## 5. 흰탭 감지 차단 + TAB-CONFIRM

### 5.1 흰탭 즉시 차단

`raw_white_only = white_raw && !red_raw`가 True인 프레임에서:
- `_whitetab_confirm += 1` (이전 프레임이 0이었으면 frame dump 1회).
- `_decide_move` 호출 안 함 → `[WHITETAB-BLOCK]` + `release_all`.
- 이 외 프레임은 `_whitetab_confirm = 0` (edge 리셋).

첫 프레임 frame dump → `logs/whitetab_suspect/whitetab_{mapneq|sameMap}_*.png`
(5초 throttle). YOLO 오검 사후 검증용.

### 5.2 TAB-CONFIRM arm 게이트

다음 모두 충족 시 `arm_tab_confirm(now, map_neq)` 호출:
- `_whitetab_confirm >= 3` (3프레임 연속 흰탭).
- `!fol._tab_confirm_active`.
- `atk.coord_valid` (ATK-COORD-JUMP 구간 회피).
- `state != DEAD && state != DISCONNECTED`.

### 5.3 Route A 단일 (Tab → 힐러 self-target 빨탭)

```
arm → Tab 송신 → A_wait_red
A_wait_red: red_raw 3프레임 연속 → done_ok
```

**Tab의 진짜 의미 (G1)**: 힐러 자기 몸에 빨탭 찍음. 이 상태로 다음 맵 진입
시 격수에게 빨탭 **자동 전이**. 즉 흰탭 감지 → Tab → 빨탭 확정 →
`is_paused=False` → `_decide_move` 재개 → exit_dir 방향키 → 포탈 통과
→ 다음 맵에서 격수 타겟 복원.

**Route B/ESC는 삭제됨 (G2)**: 격수가 NumLock ON으로 스킬 계속 시전 중 →
ESC 눌러봐야 흰탭 즉시 재등장 → 의미 없음. map_neq 분기 제거, 단일 Route A.

**흰탭 중 방향키 = 커서만 이동 (캐릭터 정지)**: `is_paused()`에
`_tab_confirm_active` 포함 유지. Route A 대기 중 `_decide_move` 호출 안 됨.
red 확정 순간 `_tab_confirm_active=False` → 정상 이동 재개.

### 5.5 TAB-CONFIRM 하드 타임아웃

10초 내 done_ok 못 만나면 `done_timeout` → 단순 취소(재송신 없음). 일반
follow 로직으로 복귀. `logs/whitetab_suspect/` 덤프 png로 YOLO 오검 사후
조사.

### 5.6 TAB-CONFIRM-CANCEL

Route 진행 중 `MAP-SEQ-EDGE` 발생 → 이전 route의 상황 전제 무효. 활성 route
즉시 취소.

### 5.7 is_paused() 게이트

`fol.is_paused(now)` = `_pause_until || _map_sync_until || _tab_confirm_active`.
True면 `_decide_move` 호출 안 되고 release_all + `MAP-PAUSE` 로그.
`tab_confirm_tick`만 호출해 Tab 송신 관리. 흰탭 상태에서 방향키 송신은 커서만
움직이고 캐릭터는 안 움직이므로 차단이 필수 (`project_tab_mechanics.md`).

## 6. trail 기록 + 진행도

맵별 `_map_trail[map]: list[(x,y)]`. 격수가 움직여 좌표가 바뀔 때마다
`TRAIL-PUSH`. 힐러가 이 좌표열을 따라가며 `progress` 갱신. 도달한
waypoint는 `reached_wps`에 기록(`PROGRESS` 로그).

오염 거부:
- 같은 맵 내 d>8 점프 → 격수 순간이동 불가 → reject.
- 새 맵 진입 첫 push가 이전 맵 exit_coord 근처 → fresh-map guard → reject.

```

```

```
[WHITETAB-ARM] confirm=3 route=A map_neq=True

[TAB-CONFIRM-DONE] route=A elapsed=XXXms map_neq_at_arm=True
```

```

```

# 알아먹기 쉽게 (시나리오 스토리)

## 캐스팅

- **격수 형님**: 인간이 조작하는 딜러. 맵을 자유롭게 이동.
- **힐러**: 자동 봇. 격수 형님 따라다니며 힐/버프.
- **UDP**: 격수 형님이 "나 여기 있어!"라고 30번/초 외치는 무전기.
- **비전**: 힐러가 자기 눈으로 보는 화면 (YOLO + OCR).

## 상황 1: 같은 맵에서 걸어다니기

격수 형님이 L로 움직임 → UDP로 "나 (140, 74)" 도착.
힐러는 자기 좌표가 (144, 75)니까 "어라 격수 형님 왼쪽이네" → L 키 누름.
`|d|`가 2 이하로 좁혀지면 키 release.

이게 `B3:coord` / `B3a:tol` 로직.

## 상황 2: 격수 형님이 포탈로 맵 이동

1. **격수 형님이 맵 경계에서 포탈 밟음** → `map_seq`가 80 → 81로 바뀜.
2. **힐러 반응**: "어 격수 형님 맵 바뀌었네" (`MAP-SEQ-EDGE`).
   - 0.25초간 키 release (갑자기 뭐하지 말고 대기).
   - 이후 1.5초간 MAP-SYNC: 내 맵 OCR이 따라잡을 때까지 보류.
3. **힐러 자기 맵 OCR이 갱신**: `healer_map == atk.map_name` → MAP-SYNC 해제.
4. **근데 힐러는 아직 원래 맵에 서 있음** (`map_neq=True`): 격수 형님은 이미
   다음 맵. 힐러는 `exit_dir`(격수가 나간 방향) 기준으로 계속 걸어가서 포탈
   통과.
5. **힐러도 포탈 통과** → `healer_map`도 새 맵으로 바뀜 → `map_neq=False` 복귀.

## 상황 3: 힐러 화면에 흰탭 떴다 (타겟 풀림)

맵이폴에서 "탭"은 타겟 고정 키. 격수가 힐러한테 박스 표시됨 = **빨탭**. 격수
타겟이 풀리면 박스가 **흰탭**(흰색) 커서로 전환.

**중요 규칙 (게임 내 물리)**:
- 흰탭 상태: 방향키 → **커서만 이동**, 캐릭터는 **정지** (제자리).
- 빨탭 상태: 방향키 → **캐릭터 이동** (정상).
- 따라서 흰탭 감지 시 방향키 송신은 무의미 + 커서 이탈 유발 → 무조건 차단.

**Tab 키의 의미**: 흰탭 상태에서 Tab → **힐러 자기 몸에 빨탭** (self-target).
이 상태로 다음 맵 진입 → 다음 맵에서 격수에게 **자동 전이**. 즉 Tab은
"격수 빨탭 복원 예약"이다.

### 조치 (Route A 단일)

1. 즉시 방향키 STOP (`WHITETAB-BLOCK`). map_neq 무관.
2. 3프레임 연속이면 arm → **Tab 키 1회** 누름.
3. 빨탭(red_raw) 3프레임 연속 → 복귀 완료 (`TAB-CONFIRM-DONE`).
4. 복귀 후 `is_paused=False` → 방향키 재개 → `exit_dir` 홀드 → 포탈 통과 →
   다음 맵에서 격수 빨탭 자동 전이.

### ESC는 왜 안 쓰나? (G2 결론 2026-04-16)

격수가 NumLock ON으로 스킬을 **계속** 시전 중. ESC로 타겟 해제해도 다음
프레임에 흰탭 즉시 재등장. 의미 없음 + 키 스팸만 남음. **Route B/ESC 삭제.**

### 상황 3-C: arm 도중 격수 형님이 맵 이동

Route A 진행 중인데 `MAP-SEQ-EDGE` 발생 → 이전 상황 전제 무효 →
`TAB-CONFIRM-CANCEL` → route 취소. 새 맵에서 흰탭 3프레임 다시 쌓이면 재 arm.

## 상황 4: 힐러가 포탈 못 밟고 고착

`trail` 끝까지 왔는데 `healer_coord`가 안 움직임 = 포탈 미스. `STAY(past)`
상태.

**조치**: 2초 기다려도 여전히 고착이면 `past_unstick` → `exit_dir` 방향으로
한 번 더 누름.

## 상황 5: 맵 OCR이 튐 (A→B→A 1초 내)

힐러 맵 OCR은 CPU라 가끔 프레임 한 번 튐. `HMAP-DEBOUNCE`가 3프레임 연속 확인
후에만 수락. 한 프레임 튄 거로 pause/sync 폭주 안 함.

## 상황 6: 격수 좌표가 말도 안 되게 튐

같은 맵인데 d=210같은 점프 → `ATK-COORD-JUMP` → coord_valid=False 강제.
격수가 이미 맵 바뀌었는데 UDP map_seq만 아직 안 온 순간.

이 상태에선 TAB-CONFIRM arm 안 함 (coord_valid 게이트에 막힘). 1초 뒤
map_seq 갱신되면 정상 플로우.

## 요약 치트시트

| 상황 | 동작 |
|---|---|
| 격수 같은 맵, 좌표 다름 | 방향키 hold |
| 격수 같은 맵, 좌표 가까움 | release |
| 격수 맵 전환 순간 | 0.25s pause + MAP-SYNC |
| 힐러 아직 이전 맵 | trail 끝까지 + exit_dir |
| 흰탭 감지 | 즉시 STOP (방향키=커서만 이동, 캐릭터 정지) |
| 흰탭 3프레임 (map_neq 무관) | Tab 1회 = 힐러 self-target → 포탈 통과 시 격수 전이 |
| 빨탭 3프레임 연속 | TAB-CONFIRM-DONE → 방향키 재개 |
| trail 끝 고착 2초 | exit_dir push |
| 격수 좌표 점프 | coord_valid=False, arm 대기 |
| 맵 OCR 튐 | 3프레임 확인 |
