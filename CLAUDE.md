## 반성문 체크리스트 (매 동작 전 자체점검)

전체 반성문: `D:\oldbaram\반성문.md`

1. 도구 호출 전 파라미터 이름 재확인 (`old_string` 같은 오타 금지).
2. **작업 환경 D:\oldbaram ≠ 실행 환경 C:\oldbaram** — 파일 수정 시 항상 "이거 사용자가 C로 직접 복사해야 함" 명시.
3. 하드코딩 기본 금지, 자동 추정 기본 선호 (해상도/창 크기 변동 대비).
4. 알고리즘 경계 조건(화면 테두리, 나무 UI 하이라이트 등)을 먼저 나열한 뒤 방지 조건 넣기.
5. 임계값·좌표 범위 같은 숫자는 **반드시 측정 후 결정**. 직관/기억 금지.
6. 확인할 수 있는 상태는 **먼저 스스로 확인** (logs 폴더 등). 사용자에게 상태 묻지 말 것.
7. **증상 우회보다 원인 수정 우선**. OCR 흔들리면 유사도 매칭 전에 입력 전처리부터 고칠 것.
8. 긴 작업 전 한 문장으로 "지금 무엇을 하고 있다" 예고.
9. 환경 변동 가능 값(IP/경로/프로세스명) 재확인. 이전 대화의 값을 "지금도 유효"로 가정 금지.
10. 수정 범위를 먼저 정리해 **한 번의 수정으로 마감** (사용자 복사 사이클 줄이기).
11. 사용자 환경 샘플(`debug_frame.png` 등)로 **로컬 선검증** 후 배포 요청.
12. 메모리 저장 지침(특히 `feedback_forbidden_approaches.md`)과 교차 점검.
13. 경로·머신 맥락 모호하면 **작업 전 한 줄 확인 질문**.
14. 명백한 안전장치 누락(예: 맵 일치 게이트)은 "추가할까요?" 묻지 말고 기본으로 추가.
15. 의도적 코드 복제 시 "합치기 작업"을 동시에 메모에 올리기.
16. 같은 주제 3회 이상 수정 중이면 어조를 겸손하게 ("아직 확실치 않습니다, 로컬 검증 후 말씀드리겠습니다").
17. "짧게"의 진짜 의미는 **정보 밀도 높게**. 테스트·복사 요청만 짧게 던지는 건 금지.
18. 송수신 역할 경계(격수=송신, 힐러=수신) 매번 의식.
19. 디스플레이 폰트는 안티앨리어싱됨 → 임계 설정 시 순백/순흑 가정 금지.
20. 사과의 반복 대신 **행동 변화**로 증명. "죄송합니다" 다음 줄에 구체적 수정 동작이 있어야 함.

- 같은 실수 반복 시 사용자가 반성문을 꺼내도 변명 금지.
