# 옛날바람 3PC 매크로 매커니즘 명세서 (v2)

> 근거: 원본 영상 170초 × 2fps = 341 프레임 전수 분석 + 사용자 사양 v2

## 0. 용어 정의

| 용어 | 의미 |
|------|------|
| 격수 | 공격 담당. 사람이 직접 조작하는 메인 PC |
| 도사 | 힐/혼마 담당. 매크로로 자동 운영되는 봇 PC 2대 |
| 빨탭 | 빠른 Tab — 타겟을 다른 유저(격수)에게 고정. 힐/혼마 대상 지정용 |
| 넘락 토글 | NumLock OFF + 넘패드 키 DOWN 유지 → 해당 키 스킬 자동 연속 시전 |
| 그룹원 | 파티 멤버 (격수 + 도사2명) |
| 체력/마력 | HP / MP |

## 1. 물리 구성

```
[격수 PC]  ← 사람 조작 (키보드/마우스 직접), 매크로 없음
[도사1 PC] ← slave.exe (원격 WAN, 로컬 자동)
[도사2 PC] ← slave.exe (원격 WAN, 로컬 자동)
```

- 3대 PC는 **WAN 원격지**. LAN 아님.
- 봇 간 상호 통신 없음. 각 PC 완전 독립.
- 게임 서버가 유일한 상태 동기점.

## 2. 격수 공격 방식

- **타겟팅이 아닌 근거리 바로 앞 공격 방식**
- 격수는 몬스터 앞에 서서 공격 스킬을 시전, 스킬이 앞 범위 몬스터에 적용
- 격수 매크로 필요 없음 — 사람이 조작
- 도사가 혼마술(방어도 +50 디버프)을 계속 걸어주면 격수 공격이 더 잘 들어감

## 3. 도사 스킬 사양

| 스킬 | 유형 | 효과 | 쿨다운 | MP 비용 | 타겟 |
|------|------|------|--------|--------|------|
| **신령의기원** | 힐 | 10,000 + (마력×3%) 회복 | 초당 3틱 | (미상, 낮음) | 타겟(격수) |
| **봉황의기원** | 힐 | 30,000 + (마력×3%) 회복 | 초당 3틱 | (미상, 중간) | 타겟(격수) |
| **백호의희원** | 자가 MP회복 | 현재 마력 50% 소모 → 300% 회복. 1초 쿨/15,000회복 | 회복량 연동 | 현재 마력 50% | 자신 |
| **백호의희원'첨** | 그룹 MP회복 | 그룹원 대상 백호의희원. 그룹원 수로 쿨 분산 | `현재 마력/10,000 / 그룹원 수`초 | 현재 마력 50% | 그룹원 |
| **혼마술** | 디버프 | 대상 방어도 +50 | (자동시전) | (낮음) | 대상 |
| **파력무참** | 버프 | 시전자 2칸 이내 유저 데미지 2배 | 180초 쿨 / 45초 지속 | (중간) | 범위 |
| **공력증강** | MP회복 | 최대마력 40%의 HP 소모 → 마력 100% 회복 | (짧음) | HP 사용 | 자신 |
| **금강불체** | 방어 | 10초 무적 (선비/흉노 원거리 마법은 못 막음. 실패 확률 有) | (미상) | (미상) | 자신 |

## 4. 도사 운영 로직 (사용자 사양 기반)

### 4.1 주력 힐 (NumLock 싸이클 상시)
- **봉황의기원** 또는 **신령의기원** — 둘 중 하나를 유저가 선택, NumLock 자동시전으로 **초당 3틱** 연속 시전
- **혼마술** — 동시에 NumLock 싸이클에 포함. 몬스터 디버프 지속
- 조건: 빨탭으로 격수 타겟 고정되어 있어야 힐이 격수에게 들어감

### 4.2 조건부 시전 스킬 (이벤트 기반, NumLock 싸이클과 별개)

| 스킬 | 시전 조건 |
|------|----------|
| 백호의희원 | 자기 MP가 임계값 이상 + 쿨 돌았을 때 (수동 쿨 관리) |
| 백호의희원'첨 | 격수 HP 낮을 때 or 쿨마다 |
| 공력증강 | 자기 MP가 임계값 이하. 단 자기 HP가 위험치면 시전 금지 |
| 파력무참 | 몬스터 존재 + 쿨 180초 완료. **도사 2명이면 겹치지 않게 오프셋** |
| 금강불체 | 옵션 (고스펙이면 OFF). ON이면 쿨마다 자기 방어 |

### 4.3 MP 관리
- 메인 힐(봉황/신령) 지속 시전 → MP 빠르게 소모
- **백호의희원**: 자가 MP 순환 회복 (50% 소모 → 300% 회복, 실질 +150%)
- **공력증강**: 응급 MP 보충 (HP 대가). **HP 안전치 이상일 때만** 사용

### 4.4 파력무참 겹침 회피 (도사 2명)
- 지속 45초 / 쿨 180초 → 하나로는 25% 유효시간
- **도사 간 통신 없음** → 타이머 오프셋 방식
- 도사1 config: `parryeokmujam_offset_sec = 0`
- 도사2 config: `parryeokmujam_offset_sec = 90`  (180초의 절반)
- 결과: 45초 시전(도사1) → 45초 off → 45초 시전(도사2) → 45초 off → 반복 (유효시간 50%)

## 5. 빨탭 (타겟 락)

### 5.1 역할
- 도사의 현재 타겟을 **격수로 고정** — 신령/봉황 힐은 타겟에게 시전
- 맵 이동 시 타겟 풀리면 힐이 허공으로 나감 → 격수 즉사 위험

### 5.2 매크로 처리
- 주기적으로 Tab을 재시전
- **격수 캐릭터명은 설치 시 GUI에서 사용자가 입력**
- 타겟 UI(상단 이름표) 이미지 매칭 또는 OCR로 현재 타겟이 격수인지 확인
- 격수 아니면 Tab 반복 → 격수 이름 잡힐 때까지

## 6. 따라가기 (Follow)

### 6.1 조건
- 도사가 **격수와 같은 맵**
- 도사가 **격수의 2칸 이내 인접** (파력무참 효과 범위 겸용)

### 6.2 화면 인식 기반
1. 도사 자기 화면에서 **격수 캐릭터 이미지 템플릿 매칭**
2. 도사 화면 중앙(= 도사 위치) 기준 상대 벡터 계산
3. 방향키 입력 이동
4. 2칸 이내 → 정지
5. 격수 사라짐 → 맵 이동 판단

### 6.3 맵 이동 대응
- 영상 관찰: 격수 포탈 진입 후 도사 창 2~4초 블랙 → 같은 맵 전환
- 매크로 흐름:
  1. 격수 사라짐 → 마지막 위치(포탈)로 이동
  2. 방향키로 포탈 진입 시도
  3. 맵 전환 감지 (화면 블랙 → 복귀)
  4. 재탐색 → 빨탭 즉시 재시행 (타겟 풀렸음)
  5. 2칸 이내 복귀

## 7. HP/MP 싸이클 (실제 관찰값)

### 7.1 격수 HP
| 상태 | HP | 주기 |
|------|-----|------|
| 풀피 | ~1,215K | 공격 직전 |
| 반토막 | 200K~400K | 격수 고위력 공격 직후 |
| 회복중 | 500K~900K | 도사 힐 1~2틱 후 |
| 풀피 복귀 | 1,200K+ | 힐 2~3틱 후 (봉황 3틱×30K=90K+ 초당) |
| 주기 | 5~7초 | 공격↔풀피 |

### 7.2 도사 MP (자체 관리)
- 봉황 지속 시전으로 점진 감소
- 백호의희원 쿨마다 순환 → 자가 유지
- 위험 시 공력증강 응급 발동

## 8. 죽음/부활 처리

### 8.1 격수 사망
- 영상에서는 발생 안 함 (힐이 잘 나갔음)
- 실제 발생 시:
  - **격수 HP 0 또는 유령 상태 감지**
  - 도사가 **부활 스킬 시전** (사양 확정 필요)
  - 이후 힐 재개

### 8.2 도사 사망
- 도사 자기 HP 0 → 유령 상태
- **자가부활 스킬 시전** (사양 확정 필요)
- 부활 후 HP/MP 충분히 회복될 때까지 대기
- 회복 완료 → INIT 복귀 → FOLLOW 상태로 루틴 재개

## 9. 동기화 불필요 — 왜 가능한가

1. **NumLock 자동시전 = 시간 기반 반복** — 격수 HP 몰라도 계속 힐
2. **오버힐은 낭비일 뿐 문제 없음**
3. **혼마/파력무참은 도사 주변 범위** — 격수 옆에 붙어있기만 하면 OK
4. **빨탭은 이미지 기반 로컬 확인** — 네트워크 불필요
5. **파력무참 겹침은 config 오프셋으로 정적 해결**

즉 **"무지성 연사 (메인힐+혼마) + 조건부 이벤트 (쿨/위험/오프셋) + 로컬 시각 추적"**.
