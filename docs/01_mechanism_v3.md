# 옛날바람 3PC 매크로 매커니즘 명세서 (v3)

> 변경: v2는 화면 템플릿 매칭 기반 따라가기 + 영상 추정이 중심.
> v3는 **메모리 리더 기반 좌표 + YOLO 빨탭 검출** 중심으로 재설계.
> 근거: 실제 구현 과정에서 확인된 사실들
>  - 빨탭이 순수 빨강이 아닌 적갈색(bloom blend) → HSV/템플릿 실패, YOLO만 유효
>  - 격수/도사 PC 모두 게임 메모리에서 맵이름+좌표 실시간 읽기 가능
>  - 점프/사다리 없음. 상하좌우 직선 이동만

## 0. 용어 정의

| 용어 | 의미 |
|------|------|
| 격수 | 공격 담당. 사람이 직접 조작하는 메인 PC |
| 도사 | 힐/혼마 담당. 매크로로 자동 운영되는 봇 PC 2대 |
| 빨탭 | 타겟이 "다른 유저(격수)"에게 걸린 상태 — 도사가 격수를 힐 대상으로 지정 |
| 흰탭 | 타겟이 "자기 자신"에게 걸린 상태 — 빨탭 풀리면 자동으로 자가 타겟 |
| 넘락 토글 | NumLock OFF + 넘패드 키 DOWN 유지 → 해당 키 스킬 자동 연속 시전 |

## 1. 물리 구성

```
[격수 PC]   사람 조작. MemReader + UDP Sender만 돌림.
[도사1 PC]  slave.exe. MemReader + UDP Sender/Receiver + YOLO + 키송신
[도사2 PC]  slave.exe. 동일.
```

- **LAN 연결 전제** (UDP 브로드캐스트 사용)
- v2에서는 "WAN 원격지"였으나, 좌표 싱크 필요로 LAN으로 변경
- UDP 손실 허용 — 최신 좌표만 덮어쓰므로 순서/누락 무관

## 2. 격수 공격 방식 (v2와 동일)

- 타겟팅 없는 근거리 앞 범위 공격
- 격수 매크로 불필요 (사람 조작)
- 도사의 혼마술(방어 +50 디버프)로 격수 공격 효율 증가

## 3. 좌표 브로드캐스트 (v3 신규)

### 3.1 송신 (격수 PC + 도사 PC 양쪽)

```
MemReader가 50Hz로 읽음:
  - 맵 이름 (map_name)
  - 캐릭터 좌표 (x, y)
  - 캐릭터 HP/MP (선택)

UDP 브로드캐스트 (port 예: 51900):
  { "role": "main"|"dosa1"|"dosa2",
    "map": "bonghwang_1",
    "x": 1234.5, "y": 567.8,
    "hp_pct": 0.72,
    "ts": 1712923456.789 }
```

### 3.2 수신 (도사 PC)

- 자기 role 제외한 다른 PC들의 상태를 최신값만 보관
- 격수 좌표 기반으로 따라가기 결정
- 다른 도사 좌표로 버프 겹침 회피 등 확장 가능

## 4. 빨탭 / 힐 타겟팅 (v3 변경)

### 4.1 역할
- 도사가 격수를 힐하려면 타겟이 격수여야 함
- Tab 눌러서 격수 타겟팅 → 빨탭 표시됨

### 4.2 검출: YOLOv8 (v2의 템플릿 매칭 폐기)
- **이유**: 빨탭 테두리가 적갈색(Bloom/AA 블렌드)이라 HSV 검출 불가능. 템플릿 매칭은 bloom/각도/스킬이펙트에 취약. 결국 딥러닝 지도학습이 유일한 경로.
- **데이터셋**: 1482장 (수동 라벨 167장 + 재학습 pseudo-label 1315장)
- **모델**: YOLOv8n, imgsz 640, FP16 추론
- **시드 학습 성과**: mAP50 0.995, mAP50-95 0.772 (47 epoch early stop)

### 4.3 힐 타겟 결정 로직
```
YOLO 추론:
  - 빨탭 bbox 검출됨 → 타겟은 격수, 주력 힐 계속
  - 빨탭 bbox 미검출 (흰탭 상태로 가정) → 자힐 모드
```
**흰탭은 별도 클래스 학습 안 함. "빨탭 미검출 == 흰탭" 룰 기반.**

## 5. 따라가기 (v3 전면 재설계)

### 5.1 v2 vs v3

| 항목 | v2 | v3 |
|------|-----|-----|
| 위치 파악 | 화면 템플릿 매칭으로 격수 캐릭터 찾기 | 게임 메모리에서 좌표 직접 읽기 |
| 방향 결정 | 화면 중앙 기준 상대 벡터 | 격수 좌표 - 도사 좌표 |
| 포털 | 화면 블랙아웃 감지 | 맵 이름 변경 이벤트 |
| 빈맵/포털 이미지 | 필요 | **불필요** |

### 5.2 상태 머신

```
SAME_MAP    격수.map == 도사.map
            → (격수.x - 도사.x, 격수.y - 도사.y)로 방향 결정
            → 2칸 이내면 정지

DIFF_MAP    격수.map != 도사.map  (격수가 먼저 맵 전환)
            → 격수의 "마지막 이동 방향"으로 계속 hold
            → 도사 맵이 바뀔 때까지 (포털 통과 대기)

ARRIVED     도사.map == 격수.map 로 복귀 → SAME_MAP
```

### 5.3 방향 결정 (SAME_MAP)
```python
dx = main.x - dosa.x
dy = main.y - dosa.y
THRESH = 2.0  # 2칸 이내면 이동 불필요 (튜닝 필요)

if abs(dx) > abs(dy):
    if abs(dx) > THRESH:
        hold(RIGHT if dx > 0 else LEFT)
elif abs(dy) > THRESH:
    hold(DOWN if dy > 0 else UP)
else:
    release_all()  # 2칸 이내 도달
```

### 5.4 맵 전환 (DIFF_MAP)
```python
# 격수 맵 바뀐 직후의 "격수 최근 이동 방향" 캐시
last_dir = 격수 좌표 변화 기록에서 최근 N 샘플 평균 방향

hold(last_dir)  # 포털 방향으로 계속 걸음

if dosa.map == main.map:
    release_all()
    transition(SAME_MAP)

# 타임아웃 5초 넘어가면 "포털 못 찾음" → 경고/정지
```

### 5.5 왜 v3가 v2보다 압도적으로 나은가
- 화면 렌더링/이펙트/카메라 스크롤에 영향 안 받음
- 빈맵/포털 위치 이미지 수집 불필요
- 좌표 오차 0 (ground truth)
- 맵 전환 판정이 deterministic (맵 이름 비교)

## 6. 스킬 운영 로직 (v2와 동일, 요약)

### 6.1 주력 힐 (NumLock 싸이클)
- 봉황의기원/신령의기원 + 혼마술 → NumLock OFF로 초당 3틱 자동 연사
- **조건**: 빨탭으로 격수 타겟팅 유지

### 6.2 조건부 스킬 (이벤트 기반)
| 스킬 | 조건 |
|------|------|
| 백호의희원 | MP ≥ 임계값 + 쿨 |
| 백호의희원'첨 | 격수 HP 낮음 or 쿨마다 |
| 공력증강 | MP 낮고 HP 안전 |
| 파력무참 | 180s 쿨, 도사 간 90s 오프셋으로 겹침 회피 |
| 금강불체 | 옵션, 쿨마다 |

### 6.3 HP/MP 체크 방식 (v3 변경)
- v2: 화면 HP/MP 바 픽셀 읽기 + 그룹창 격수 HP
- v3: **메모리 리더가 직접 HP/MP 읽음** → UDP로 broadcast된 격수 HP% 사용
- 픽셀 읽기는 백업/검증용으로 유지

## 7. 흰탭 처리 (v3 신규)

### 7.1 흰탭 발생 조건
- 맵 전환 직후 파티 타겟 풀림
- 격수 사망 / 격수 이탈
- 도사 자신이 파티 시야 밖

### 7.2 대응
```
빨탭 미검출이 1초 이상 지속
  → 흰탭 상태로 판정
  → 주력 힐 싸이클은 자기 자신에게 들어감 (자힐 효과)
  → 이동은 계속 (DIFF_MAP이면 포털 방향 유지)
  → 빨탭 재검출되면 싸이클 자동 복귀
```

## 8. 사망/부활 (v2와 동일)
- 자기 HP 0 / 유령 감지 → 자가부활 → 회복 대기 → INIT
- 격수 사망 감지 → 부활 스킬 (옵션)

## 9. 동기화 불필요 → 최소 동기화로 변경 (v3)

| 측면 | v2 | v3 |
|------|-----|-----|
| 봇 간 통신 | 없음 | UDP 좌표 브로드캐스트 (최소) |
| 파력무참 오프셋 | config 정적 | 그대로 유지 |
| 게임 서버 동기점 | 유일 동기점 | 여전히 권위. 브로드캐스트는 보조 |

UDP 누락/지연은 치명적이지 않음 — 다음 패킷이 바로 덮어씀.
